# 因果认知地图 · 融合引擎 Spec

## Overview
- **Summary**: 在 Alethia AI 知识管理系统 v0.5 中新增因果认知地图引擎，实现认知地图与知识图谱的双向融合、自由嵌套视图、AI 自然语言操控、因果推理引擎、Markdown 因果语法、证据链护栏、版本化与对比、实时预警及反事实叙事。
- **Purpose**: 让知识库从"静态归档"升级为"动态推理与决策支持系统"，打通定性因果逻辑与定量事实数据。
- **Target Users**: 知识库管理员、分析师、决策者、企业战略用户

## Why
当前知识库仅存储事实和关系，缺乏因果推理、假设推演和动态结构化视图能力。用户无法在知识库中回答"如果…会怎样"类问题，也无法将散落的知识图谱节点组织成因果认知地图。

## What Changes

### 第一层：因果认知地图 Markdown 语法（L3 解析层）
- 新增 `## Causal Model` 区块语法，支持因果声明、贝叶斯 CPT 表
- 解析器扩展识别因果边，存入 `causal_edges` 缓存表
- `rebuild-struct` 时重建因果图

### 第二层：因果认知地图可视化（前端视图层）
- 基于 G6/Cytoscape.js 实现因果认知地图画布
- 自由嵌套：打包/解包、透视模式、多分支展开
- 节点与知识图谱实体双向绑定（变量 ↔ 实体/KPI 属性）

### 第三层：AI 自然语言操控（交互层）
- LLM 解析自然语言指令转化为图操作 API
- 智能建议：社区检测、路径分析、预设视图推荐
- 跨图谱融合定位

### 第四层：因果推理引擎（推理层）
- 贝叶斯网络概率推理 + 时间脉冲响应模拟
- 反事实推理与回溯推演
- 增量学习与参数更新

### 第五层：证据链与护栏（安全层）
- 因果边证据溯源
- 推理置信度呈现
- 人类审核与干预流程

### 第六层：版本化与对比（版本控制层）
- Git 式模型版本树
- 分歧对比视图
- 回滚保护

### 第七层：实时预警（感知层）
- 因果预警链路
- 空间与物理世界映射

### 第八层：自解释与叙事（展示层）
- 自然语言反事实段落
- 因果推演视频导出
- 假设前提显式化与挑战

### 第九层：视图-真相解耦（架构层）
- 视图状态独立于知识真相
- 前端虚拟化渲染
- 后端异步增量计算

## Impact
- Affected code: 新建 `server/src/causal/`、`web/src/components/CognitiveMap/`、`web/src/routes/CognitiveMapPage.tsx` 等
- 新增数据库迁移：`causal_edges` 表、`causal_cpt` 表
- 新增 API 路由：`/api/causal/` 系列
- 新增 Markdown 解析器扩展

## ADDED Requirements

### Requirement: 因果 Markdown 语法
系统 SHALL 解析实体页面中 `## Causal Model` 区块，提取因果声明存入 `causal_edges` 表。

#### Scenario: 解析因果声明
- **WHEN** `rebuild-struct` 扫描到实体页面含 `## Causal Model`
- **THEN** 每行因果声明解析为 `(source, target, relation, lag, weight, conf)` 存入缓存

### Requirement: 因果认知地图画布
系统 SHALL 提供可视化画布，支持节点拖拽、因果边渲染、自由打包/解包/透视。

#### Scenario: 打包节点
- **WHEN** 用户选中多个节点并右键"打包成新节点"
- **THEN** 生成虚拟节点，双击可展开内部结构

### Requirement: AI 自然语言操控
系统 SHALL 支持自然语言指令操控视图（打包/展开/过滤/透视）。

#### Scenario: 自然语言打包
- **WHEN** 用户说"把供应商风险相关的所有节点打包成一个节点"
- **THEN** AI 解析意图，执行选择、打包操作

### Requirement: 因果推理引擎
系统 SHALL 支持贝叶斯网络概率推理、反事实推理和回溯推演。

#### Scenario: 干预推理
- **WHEN** 用户问"如果提高自动化覆盖率，运营效率变高的概率是多少"
- **THEN** 引擎计算 P(目标=高 | do(干预=值))，返回概率变化和置信区间

### Requirement: 证据链与护栏
系统 SHALL 为每个因果声明记录证据来源，推理输出附带置信区间和假设前提。

### Requirement: 版本化与对比
系统 SHALL 支持因果模型版本提交、分支和分歧对比。

### Requirement: 实时预警
系统 SHALL 支持因果边阈值预警，父节点状态变化时推送通知。

### Requirement: 视图-真相解耦
系统 SHALL 将所有视图状态（打包、展开、透视）作为独立于 Markdown 知识真相的临时缓存。
