# 理想 AI 知识库 v5.0 认知共生版 · Web 前端界面设计文档

**版本**：v1.1（审计修正版）  
**日期**：2026-07-03  
**对应架构**：v5.0  
**设计原则**：全汉化无障碍 · 人类掌权 · 认知共生 · 无代码操作 · 可传世

---

## 1. 设计概览与全局规范

### 1.1 设计语言

- **视觉基调**：沉静、信赖、智慧。以深色石板灰与羊皮纸暖白为主色调，点缀知识蓝与共生绿。支持浅色/深色双主题，默认跟随系统，可在设置中手动切换。
- **排版**：正文采用高可读性的无衬线字体（Inter、思源黑体），代码与 Markdown 源文本使用等宽字体（JetBrains Mono）。
- **图标系统**：统一使用 Phosphor Icons，与操作语义严格对应（例如：`Brain` 代表 AI 操作，`GitFork` 代表版本，`Translate` 代表双语证据）。
- **动效**：微交互反馈（按钮悬停缩放、卡片展开过渡），避免过度动画；危险操作使用警示色+二次确认。
- **无障碍**：所有交互元素具备清晰的 focus 样式，支持键盘导航，提供 ARIA 标签，色彩对比度符合 WCAG AA 级。为图标按钮提供 tooltip 或 aria-label。屏幕阅读器可正确获取动态内容更新。

### 1.2 全汉化策略

- **界面语言**：默认加载 `zh-CN` 语言包，通过 `react-i18next` 实现文案统一管理。除技术专有名词（如 API、Markdown、Diff）外，所有文案均为中文。
- **系统消息**：后端返回的信息若含英文，前端根据语言设置自动映射为中文；未映射的保留原文并附带提示。
- **证据双语**：所有 `evidence_span` 弹出层默认展示中文译文，原文折叠。非中文源语言展示 “译” 图标。

### 1.3 布局系统

采用可复用的**全局壳层**（Shell）包裹所有页面。全局壳层包含：

- **顶栏**：全局搜索框、快捷 AI 问答入口、预算快览、系统消息中心、用户头像菜单。
- **左侧导航栏**：可折叠，宽度 240px。包含主导航链接、快速操作（新建页面、触发维护）、当前状态提示点（审核队列数量、幽灵关系数）。
- **主内容区**：动态路由出口，内边距 24px，最大宽度 1440px 居中。
- **底部状态栏**（可选）：显示最近系统事件、后台任务进度、最后索引时间。

移动端适配：≤768px 时左侧导航转为底部标签栏，顶栏搜索折叠为图标，主内容区全宽。

### 1.4 组件命名与组织

所有组件遵循 **Atomic Design** 轻度改造：

- `ui/`：原子组件（按钮、输入框、标签、图标、工具提示）
- `blocks/`：分子/有机体组件（证据弹窗、Diff 卡片、图谱节点卡片、消息气泡）
- `features/`：功能页面级组件（问答面板、审核面板、仪表盘），按路由模块划分
- `layouts/`：全局壳层、设置布局等

组件命名采用 PascalCase，与文件名一致。每个组件包含独立的样式模块（CSS Modules）或 Tailwind 类。

### 1.5 深色/浅色主题

- 在 `tailwind.config.js` 中定义 `dark` 模式变量，通过 HTML 根元素的 `class="dark"` 切换。
- 设置页“外观”项提供：跟随系统、浅色、深色三个选项。选择后即时生效并持久化到 localStorage。
- 所有组件均使用语义化的颜色令牌（如 `bg-primary`, `text-secondary`），确保两种主题下可读性。

### 1.6 新用户引导

首次进入应用时，展示一个 3-5 步的引导浮窗（可跳过）：

1. **欢迎**：介绍知识库的核心理念（人机共生的数字图书馆）。
2. **百科浏览**：指向首页的搜索框和领域导航。
3. **AI 问答**：示范如何向知识库提问。
4. **审核与进化**：简介 Diff 审核的作用，以及如何让 AI 学习。
5. **完成**：引导至首页，提示可随时在帮助菜单中重新查看。

每个重点功能页面（图谱、审核、问答、仪表盘）在空状态时显示情境化的引导文案，例如审核面板空时：“还没有 AI 生成的建议。当你上传文件或让 AI 学习后，这里会出现待审核的变更。”

## 3. 全局壳层与通用组件

**幽灵关系指示**：当系统存在幽灵关系时，审核中心图标旁额外出现一个红色小幽灵图标 (`Ghost`)，悬停显示数量，点击可跳转至审核面板 🟢 区（幽灵清理建议）。

### 3.3 通用消息中心 (NotificationCenter)

浮层面板，分组展示：

- **审核提醒**：“3 条新的 🟡 预览区 Diff 等待您审核”
- **系统事件**：“夜间任务已成功完成” / “夜间任务因预算熔断已暂停”
- **补提取观察**：“‘经济学讲义.pdf’ 已被引用 5 次，是否需要提取？”
- **异常告警**：“影子评估发生异常熔断，请检查”

每条消息右侧有操作按钮（如“前往审核”、“立即提取”、“查看详情”），点击后跳转相应页面并清除该消息。

### 3.4 底部状态栏 (StatusBar)

默认显示在 App 底部（可配置隐藏），提供：

- 后台任务进度条（例如：“正在重建索引 67%...”）
- 知识库最后更新时间（“最后更新：10 分钟前”）
- 服务连接状态指示灯（绿点在线，红点离线）

## 4. 核心页面详细设计

### 4.1 Wiki 首页 (`/`)

#### 布局
```
┌─────────────────────────────────────────────────────────┐
│  Hero 区域 (欢迎语 + 大搜索框 + 热门入口)                │
├─────────────────┬───────────────────────────────────────┤
│ 左侧 (2/3)      │ 右侧 (1/3)                           │
│                 │                                       │
│ 📌 精选条目卡片  │ 📰 最近动态 (Timeline Feed)          │
│ (轮播/静态)     │ (限5条，滚动查看)                     │
│                 │                                       │
│ 🧭 知识领域网格  │ ⚡ 快捷操作                           │
│ (portals 卡片)  │  · 随机条目                           │
│                 │  · 知识缺口一览                       │
│                 │  · 今日问答历史                       │
└─────────────────┴───────────────────────────────────────┘
```

#### 组件分解

**HeroSection**
- 动态问候语：“下午好，[用户名]。你的知识库今天新增了 3 条问答，2 个概念正在演化。”
- 大号 `GlobalSearch` 组件，下方两个按钮：“搜索百科”和“直接向 AI 提问”。

**FeaturedArticleCard**
- 可配置的精选条目。系统根据 `quality: featured` 或最近修改频率自动推荐。显示标题、简短摘要、标签、最后更新时间。
- 手动置顶：管理员可在条目页将其“设为精选”。

**PortalGrid**
- 自动读取 `wiki/portals/` 下的 Markdown 文件，生成网格卡片。每张卡片显示门户名称、简介、成员数量。点击进入 `/wiki/portals/science`。

**TimelineFeed**
- 聚合所有条目的 Timeline 条目，仅显示公开的（`🗣 问答`、`版本变更`、`年轮生成`、`文件补提取`等）。
- 每条动态左侧有图标标识类型，右侧有时间戳（相对时间）。点击可跳转至具体条目或对话。
- 支持筛选：“只看问答动态”、“只看知识演化”。
- 底部“查看全部时间线”链接跳转至 `/timeline`。

**QuickActions**
- 按钮组：“随机漫步”（随机跳转一个条目）、“知识缺口”（跳转至仪表盘缺口分析卡片）、“今日问答”（跳转至 `/qa` 并列出今日会话）。

#### 交互
- 页面初次加载时，Hero 区域显示简短的骨架屏动画。
- 动态流支持无限滚动，或点击“查看更多”加载至独立时间线页。

### 4.2 条目详情页 (`/wiki/*`)

#### 布局
```
┌──────────────────────────────────────────────────────┐
│ 面包屑导航：首页 > 分类 > 当前条目                     │
├───────────────────────────┬──────────────────────────┤
│ 主内容区 (宽度 70%)        │ 右侧上下文侧栏 (30%)      │
│                           │                          │
│ 标题 + 多语境标签          │  🧠 迷你知识图谱         │
│                           │  (Cytoscape 局部渲染)    │
│ 渲染后的 Markdown 正文     │                          │
│ (含 evidence 角标悬停)     │  📅 条目时间线 (最近5条) │
│                           │                          │
│ 底部区块：                │  🔗 相关条目列表          │
│  · 版本历史 (折叠)        │                          │
│  · 语义年轮 (折叠)        │  💡 操作区               │
│  · 关系列表               │  · “向 AI 提问此条目”    │
│                           │  · “查看文件溯源”        │
│                           │  · “编辑此条目” (跳转    │
│                           │    至在线编辑器)          │
└───────────────────────────┴──────────────────────────┘
```

#### 主内容区核心组件

**MultiContextBadge**  
- 在标题下方以标签形式展示该实体的所有语境，如 `物理学` `信息论`。标签可点击，跳转至该语境过滤下的图谱或搜索。

**MarkdownRenderer**  
基于 `markdown-it` 的定制渲染器，实现：
- **媒体原地嵌入**：`![描述](library://hash#t=10s)` 自动转换为 `<brain-media>` 组件。视频/音频显示播放器，PDF 显示预览卡片，图片直接懒加载。
- **Evidence 角标**：`[^span_id]` 渲染为上标数字角标。悬停或点击时弹出 `EvidencePopover` 浮层。
- **历史区块折叠**：`## Version History` 和 `## Semantic Rings Archive` 默认折叠，可展开。归档链接（`[[changelog/entropy]]`）可点击跳转至归档文件视图。
- **Wiki 链接**：`[[关系实体]]` 渲染为可点击的内部链接，跳转至对应条目。
- **代码高亮**：使用 Shiki 在客户端高亮。

**EvidencePopover**  
- 弹出位置自适应，内容包含：
  - **默认显示中文译文**（若存在），并带有“译”图标提示。
  - 原文折叠在下方，点击“查看原文 (English)”展开。
  - 原文来源定位：`来源：经济学讲义.pdf，第 23 页`，可点击跳转至 `library/:hash` 页面并定位。
  - 复制引用：复制 `library://hash#page=23` 链接。

#### 右侧侧栏组件

**MiniKnowledgeGraph**  
- 以当前实体为中心，渲染 1-2 跳内的关系图。节点可拖动，点击节点跳转。
- 幽灵关系用红色虚线标记，悬停显示“目标实体已删除”。
- 数据通过 `GET /api/graph/{slug}?depth=2` 获取，前端缓存。

**EntryTimeline**  
- 提取该条目 Timeline 最近 5 条，样式与首页动态一致。点击“查看全部”可跳转至 `/timeline?entity=...` 完整时间线。

**RelatedEntries**  
- 列出 `Relations` 中的实体，按类型分组（人物/概念/文件等）。

**QuickActions**  
- “就此条目提问”：点击后跳转至 `/qa`，并自动将条目上下文（slug）作为对话的初始系统消息注入（通过 BrainAPI `contextEntity` 参数）。

#### 响应式
- 宽度 ≤1024px 时右侧侧栏消失，改为底部可展开抽屉。
- 移动端 Markdown 媒体自动适配宽度。

### 4.3 全屏知识图谱 (`/graph`)

全屏沉浸式视图，顶栏和侧栏可隐藏（按 `F` 键切换全屏）。

**画布**
- 使用 Cytoscape.js 渲染，节点样式：
  - 概念（紫色圆形）、人物（橙色方形）、文件（青色菱形）
  - 文件节点颜色表示提取状态（灰=新、蓝=部分、绿=完成）
  - 大小反映关联边数量
- 边样式：实线表示确认关系，虚线表示推断关系，红色虚线表示幽灵关系。
- **集群框**：根据夜间社区检测结果，用半透明圆角矩形包裹同一社区节点，框颜色反映生命周期（青=新兴、绿=稳定、橙=衰变、灰=归档）。

**工具栏**
- 顶部浮动工具栏：
  - 布局切换：力导向 / 同心圆 / 树形 / 层级
  - 搜索节点：输入实体名，高亮定位并居中
  - 筛选：按实体类型、置信度、最后更新时间过滤
  - 导出图片（PNG/SVG）

**时间推演滑块**
- 底部的时间轴组件，范围从知识库最早 Timeline 条目到当前时间。
- 拖动滑块时，图谱动态显示该时间点存在的节点和边，晚于该时间的隐藏。
- 滑块旁有播放按钮，自动按时间推进展示知识增长过程。
- 数据通过后端聚合的时间片接口提供（`GET /api/graph/timeslices`），避免前端全量计算。

**节点交互**
- 点击节点：显示摘要卡片，包含标题、简短描述、语境标签。双击跳转条目页。
- 右键菜单：直接提问、查看溯源、设为精选、隐藏等。
- 幽灵关系边点击后弹出提示：“目标实体 ‘旧理论’ 已不存在，建议清理此关系”，并提供“去清理”按钮跳转审核面板。

**图例与统计**
- 左下角浮动图例，展示节点/边颜色意义。
- 右下角显示当前可见节点/边总数。


#### Diff 卡片

每个卡片显示：
- 类型标签（`State变更`、`关系新增`、`幽灵清理` 等）
- 影响实体名称 + 简短变更描述
- 置信度分数和交叉验证分（🟡/🔴区显示）
- 来源文件（如果有）
- 操作按钮：`合并` / `拒绝` / `稍后处理`

**🟢 批量区**：卡片顶部有“全选”复选框，支持多选后一键批量合并或拒绝。幽灵清理建议会集合成一个单独的“清理所有幽灵关系”组，可一键处理。

**🟡 预览区**：必须逐条预览，但允许全选后一次性合并/拒绝（需二次确认）。

**🔴 重点区**：无批量操作，每条 Diff 必须点开详细对比，并手动确认合并或拒绝。拒绝时需选择原因（如“信息来源不足”、“与已有知识冲突”）。

#### 详细对比视图 (选中某 Diff 时显示)

- **并排对比**：左侧渲染变更后的富文本预览（包含媒体、证据角标），右侧展示原始 Markdown 的 diff 高亮（使用 `react-diff-viewer` 或自研组件）。
- **原文引用**：若 Diff 包含 `evidence_span`，展示对应的原文文本（双语），并附 `library://` 链接。
- **影响范围**：此变更会联动影响哪些实体/关系（通过图谱查询），以列表形式展示。
- **操作确认**：底部浮动操作栏，包含 `合并` / `拒绝` 按钮，以及一个“同时查看下一个”的快捷开关。

#### 键盘快捷键

- `J` / `K`：上下切换 Diff 卡片
- `Enter`：展开详情
- `Shift+Enter`：合并当前项
- `Backspace`：拒绝当前项

#### 审核历史

侧边栏可切换至“已审核历史”，查看最近 30 天内的审核记录，支持按操作者（人类/系统）、结果（合并/拒绝）筛选。

## 5. 关键通用组件详解

### 5.1 EvidencePopover (证据双语弹窗)

**用途**：在所有出现 `[^span_id]` 的地方（条目正文、问答消息、Diff 对比视图）复用。

**功能**：
- 触发方式：鼠标悬停 200ms 后出现，点击钉住（pin）。
- 内容：
  - 顶部：来源文件名称 + 位置（如“第 23 页”或“12:05”），可点击跳转文件预览。
  - 主体：**优先展示中文译文**（若存在译文）。译文前有 `译` 标签，悬停显示翻译模型名称。
  - 原文折叠在下方，通过“查看原文 (English)”展开。
  - 底部按钮：`复制 library:// 链接`、`在文件预览中打开`。
- 样式：白色卡片（浅色主题）/ 深灰卡片（深色主题），阴影，最大宽度 400px。与触发元素对齐（使用 Floating UI）。

**空状态**：若证据正在异步翻译中，显示“翻译中...”的加载指示器。

### 5.2 BrainMedia Web Component

**标签**：`<brain-media>`

**属性**：
- `src`：`/api/library/:hash` URL
- `type`：`video` / `audio`
- `start`：起始时间秒数
- `title`：媒体描述
- `poster`：视频封面图（可选）

**功能**：
- 自动跳转到 `start` 时间点。
- 控制栏：播放/暂停、时间显示、音量、全屏、下载按钮（若权限允许）。
- 右侧引用角标：点击复制 `library://` 链接。
- 响应式宽高，最大宽度 100%。

**实现**：使用 Custom Elements API，打包为独立 JS 文件，在页面中直接使用。

### 5.3 MarkdownRenderer 与插件

**主渲染器**：
- 初始化 `markdown-it`，开启表格、脚注、缩写等插件。
- **自定义插件 1：brain-media 嵌入**  
  规则：匹配 `!\[.*?\]\(library:\/\/[a-f0-9]+(#.*?)?\)`，解析哈希和片段，查询 MIME 类型，输出对应的 HTML 标签。
- **自定义插件 2：evidence 角标**  
  规则：将 `[^span_id]` 转换为 `<sup class="evidence-ref" data-span-id="...">[span_id]</sup>`，后续由 React 组件挂载事件。
- **自定义插件 3：折叠区块**  
  规则：将 `## Version History` 和 `## Semantic Rings Archive` 标题后的内容包裹在 `<details>` 中，默认折叠。
- **自定义插件 4：归档链接**  
  规则：将 `[[changelog/...]]` 转换为指向 `/changelog?file=...` 的链接。

**在 React 中使用**：渲染后通过 `ref` 获取 DOM，使用 `useEffect` 绑定证据角标的点击事件，弹出 `EvidencePopover`。

### 5.4 Diff 并排对比组件

- 基于 `diff` 库计算文本差异。
- 左侧面板：渲染“新文本”的 Markdown 预览（通过 `MarkdownRenderer`）。
- 右侧面板：逐行对比原始文本与新文本，用绿色/红色背景标记增删。
- 支持同步滚动（左右联动）。
- 对于 `evidence_span` 相关变更，在行尾提供角标，悬停可查看证据。

### 5.5 预算与成本显示组件

**BudgetBadge** (顶栏)：
- 圆形进度环 SVG，百分比为 `今日费用 / 日预算`。
- 颜色：<50% 绿色，50-80% 黄色，>80% 红色。
- 悬停显示具体金额和预估剩余天数。

**CostGauge** (仪表盘)：
- 水平进度条，分段显示不同任务类型的费用占比（提取、问答、翻译等），每种颜色对应图例。
- 鼠标悬停段落显示具体费用。

**CostEstimator** (问答输入区旁)：
- 在用户输入问题时，实时预估本次提问将消耗的费用（基于问题长度和历史均值），显示微小文本：“预估费用 ~$0.02”。

## 6. 状态管理与数据流

### 6.1 全局状态 (React Context)

使用 Context 分割关注点：

- `AuthContext`：用户 Token、登录状态、用户信息。
- `SettingsContext`：全局配置（语言、主题、预算状态、启用功能），从后端加载并缓存。
- `NotificationContext`：消息中心列表，未读计数，提供添加/清除方法。
- `BrainAPIContext`：API 调用实例和基础配置（baseURL、headers）。

### 6.2 服务端状态 (TanStack Query)

所有从 BrainAPI 获取的数据（条目、图谱、Diff 列表、问答对话、仪表盘指标等）均使用 React Query 管理：

- 每个 API 端点对应一个 `queryKey`，如 `['entry', slug]`，`['dashboard']`。
- 变更操作（审核合并/拒绝、回滚、设置保存）使用 `useMutation`，成功后失效相关查询缓存。
- 问答对话使用无限查询（`useInfiniteQuery`）加载历史消息。
- 实时性要求高的数据（如后台任务进度）使用 WebSocket 或轮询（`refetchInterval`）。

### 6.3 本地状态

- 表单临时状态（设置页、审核操作）使用组件内部 `useState` 或 `useReducer`。
- UI 交互状态（侧栏折叠、弹窗显示）同样内部管理。

## 7. 路由与认证守卫

- 使用 `react-router-dom` v6，配置路由表。
- 所有需要认证的路由包裹在 `<ProtectedRoute>` 组件中，检查 `AuthContext` 中的 Token。无 Token 时重定向至 `/login`。
- 登录页（简单表单，输入 API Key 或用户名密码，Phase 1 仅 API Key）。登录成功后存储 Token 到 `localStorage` 并刷新全局状态。
- 首次登录后若未完成引导，自动跳转 `/onboarding`。

## 10. 性能优化

- **代码分割**：每个路由页面使用 `React.lazy` + `Suspense` 动态加载。
- **图谱优化**：大量节点时使用虚拟化或 WebGL 渲染器（Cytoscape 支持），布局计算放入 Web Worker。
- **Markdown 渲染优化**：长文本虚拟滚动（使用 `react-window`），媒体懒加载，图片模糊占位。
- **缓存策略**：`React Query` 的 `staleTime` 根据数据特性设置（条目详情 5 分钟，仪表盘 30 秒，设置永久新鲜）。
- **Bundle 分析**：定期使用 `rollup-plugin-visualizer` 优化打包体积。

## 11. 安全防护

- **XSS 防护**：Markdown 渲染后使用 DOMPurify 清洗，不允许任意脚本。
- **CSRF 防护**：API 使用 Bearer Token，无 Cookie 依赖。
- **内容安全策略 (CSP)**：通过 Meta 标签或服务器响应头限制资源来源。
- **敏感信息保护**：所有 API 密钥、密码字段在前端使用 `type="password"` 遮罩，且不在 URL 中传递。
- **静态站点适配**：导出静态站点时，所有需要 API 调用的功能（问答、搜索、实时数据）被降级为只读展示或移除。动态组件（如搜索框）替换为静态索引跳转。图谱使用 vis-network 静态库，证据翻译数据内嵌，问答入口替换为“静态问答日志”页面。详见架构文档附录 C。

## 14. 静态站点降级适配

当用户执行 `brain generate-static-site` 后，前端项目需配合生成静态版本。设计要点：

- 构建时通过环境变量 `VITE_STATIC=true` 启用静态模式。
- 所有 API 请求替换为静态 JSON 文件读取（如 `search-index.json`、`graph.json`）。
- 问答面板替换为“问答日志”只读页面，展示历史问答记录。
- 设置页、审核面板、评估报告等仅保留只读查看入口。
- `<brain-media>` 组件指向 `./media/` 相对路径。
- 图谱使用 vis-network 静态渲染，读取 `graph.json`。
- 搜索使用客户端 Lunr.js 或 MiniSearch。

该模式确保知识库脱离服务端后仍可被完整浏览，真正实现“可传世”。

## 15. 无障碍与开发规范补充

- 所有图标按钮均设置 `aria-label`，或附带可见文字标签。
- 模态框弹出时，焦点自动移至对话框，关闭后还原。
- 动态内容更新使用 ARIA live region 通知屏幕阅读器（如新消息 toast）。
- 拖拽配置需支持键盘操作替代方案（提供下拉选择器作为降级）。
- 颜色不是传达信息的唯一方式，关键状态同时使用图标和文本。
- 开发阶段引入 axe-core 或 Lighthouse 进行自动化无障碍检查。
